Knowledge Bases

KnowledgeDBs are isolated containers for your documents and images. Each knowledge base has its own namespace, enabling multi-tenant document management and scoped AI search.

All KnowledgeDB endpoints require authentication. Include Authorization: Bearer <idToken> in your request headers.
POST

Create KnowledgeDB

/knowledgedbs

Create a new knowledge base. Each KnowledgeDB gets a unique namespace scoped to your user account. Names must be unique per user.

Request Body
ParameterTypeDescription
namestring requiredUnique name for the knowledge base.
descriptionstring optionalDescription of the knowledge base.
Response Fields
FieldTypeDescription
knowledgeDbIdstringUnique identifier (UUID) for the created knowledge base.
namestringName of the knowledge base.
descriptionstringDescription provided.
namespacestringInternal namespace: {userId}__{knowledgeDbId}.
messagestringSuccess confirmation.
// POST /knowledgedbs
// Header: Authorization: Bearer <idToken>
{
  "name": "test-docs-2",
  "description": "Test document collection"
}
// 201 Created
{
  "knowledgeDbId": "281e85f4-00b8-4613-b518-a3b4679e8d4c",
  "name": "test-docs-2",
  "description": "Test document collection",
  "namespace": "f4789468-7021-...281e85f4-00b8-...",
  "message": "KnowledgeDB created successfully"
}
# cURL example
curl -X POST \
  https://fgbpqt2pq6.execute-api.us-east-1.amazonaws.com/prod/knowledgedbs \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <idToken>" \
  -d '{ "name": "my-docs", "description": "My document collection" }'
GET

List KnowledgeDBs

/knowledgedbs

Retrieve all knowledge bases belonging to the authenticated user. Returns an array of KnowledgeDB objects with metadata including document counts.

Response Fields
FieldTypeDescription
knowledgedbsarrayArray of KnowledgeDB objects.
countnumberTotal number of knowledge bases.
KnowledgeDB Object
FieldTypeDescription
knowledgeDbIdstringUnique identifier.
namestringKnowledge base name.
descriptionstringDescription.
createdAtstringISO 8601 creation timestamp.
documentCountnumberNumber of documents in this knowledge base.
namespacestringInternal namespace identifier.
// GET /knowledgedbs
// Header: Authorization: Bearer <idToken>
// 200 OK
{
  "knowledgedbs": [
    {
      "knowledgeDbId": "281e85f4-00b8-4613-b518-a3b4679e8d4c",
      "name": "test-docs-2",
      "description": "Test document collection",
      "createdAt": "2025-07-04T11:51:59.000000",
      "documentCount": 3,
      "namespace": "f4789468-...281e85f4-..."
    }
  ],
  "count": 1
}
DELETE

Delete KnowledgeDB

/knowledgedbs/{knowledgeDbId}

Permanently delete a knowledge base and all its associated documents, images, and vector embeddings. This action is irreversible.

Path Parameters
ParameterTypeDescription
knowledgeDbIdstring requiredID of the knowledge base to delete.
Response Fields
FieldTypeDescription
messagestringConfirmation message with the deleted KB name.
deletedDocumentsnumberNumber of S3 objects deleted.
This permanently deletes the knowledge base, all documents, all images, and all indexed vector data. This cannot be undone.
// DELETE /knowledgedbs/{knowledgeDbId}
// Header: Authorization: Bearer <idToken>
// 200 OK
{
  "message": "KnowledgeDB 'test-docs-2' and all associated documents deleted successfully",
  "deletedDocuments": 6
}
