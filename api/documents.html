Documents

Upload files or generate AI-written documents into your knowledge bases. Documents are automatically indexed by Amazon Bedrock Knowledge Base for RAG search. Supported formats: PDF, DOCX, TXT, Markdown.

All document endpoints require authentication. Include Authorization: Bearer <idToken> in your request headers.
After uploading or generating a document, Bedrock Knowledge Base automatically processes and indexes it. This happens asynchronously — the document status will show "processing" until indexing completes.
POST

Upload Document

/documents/upload

Upload a file to a knowledge base. Uses multipart/form-data encoding. Maximum file size is 10MB. Supported formats: PDF, DOCX, TXT, MD.

Form Fields
FieldTypeDescription
filefile requiredThe document file to upload.
knowledgeDbIdstring requiredTarget knowledge base ID.
Response Fields (201)
FieldTypeDescription
documentIdstringUnique document identifier.
filenamestringOriginal filename.
chunkCountnumberInitially 0, updated after indexing.
messagestringSuccess confirmation.
 
// POST /documents/upload
// Content-Type: multipart/form-data
// Header: Authorization: Bearer <idToken>

// Form fields:
file: <binary file data>
knowledgeDbId: "281e85f4-00b8-4613-b518-a3b4679e8d4c"
// 201 Created
{
  "documentId": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "filename": "technical-guide.pdf",
  "chunkCount": 0,
  "message": "Document uploaded and processing started. Ingestion is handled automatically by Bedrock Knowledge Base."
}
# cURL example
curl -X POST \
  https://fgbpqt2pq6.execute-api.us-east-1.amazonaws.com/prod/documents/upload \
  -H "Authorization: Bearer <idToken>" \
  -F "file=@./document.pdf" \
  -F "knowledgeDbId=281e85f4-00b8-4613-b518-a3b4679e8d4c"
POST

Generate Document

/documents/generate

Generate a markdown document using AI (Claude Haiku 4.5) based on a text prompt. The generated document is automatically stored in S3 and indexed for search.

Request Body
ParameterTypeDescription
promptstring requiredTopic or instructions for document generation.
knowledgeDbIdstring requiredTarget knowledge base ID.
Response Fields (201)
FieldTypeDescription
documentIdstringUnique document identifier.
filenamestringAuto-generated filename from prompt.
promptstringOriginal prompt used.
contentPreviewstringFirst 500 characters of generated content.
messagestringSuccess confirmation.
 
// POST /documents/generate
// Header: Authorization: Bearer <idToken>
{
  "prompt": "Write a guide about microservices architecture",
  "knowledgeDbId": "281e85f4-00b8-4613-b518-a3b4679e8d4c"
}
// 201 Created
{
  "documentId": "b2c3d4e5-f6a7-8901-bcde-f12345678901",
  "filename": "write-a-guide-about-microservices-architecture.md",
  "prompt": "Write a guide about microservices architecture",
  "chunkCount": 0,
  "contentPreview": "# Microservices Architecture Guide\n\n## Introduction...",
  "message": "Document generated and stored successfully"
}
GET

List Documents

/documents

List all documents for the authenticated user. Optionally filter by knowledge base ID using a query parameter.

Query Parameters
ParameterTypeDescription
knowledgeDbIdstring optionalFilter documents by knowledge base.
Document Object
FieldTypeDescription
documentIdstringUnique identifier.
filenamestringOriginal or generated filename.
fileTypestringMIME type (e.g., application/pdf).
fileSizenumberFile size in bytes.
knowledgeDbIdstringParent knowledge base ID.
chunkCountnumberNumber of indexed chunks.
createdAtstringISO 8601 timestamp.
statusstringprocessing, indexed, or reprocessing.
 
// GET /documents?knowledgeDbId=281e85f4-...
// Header: Authorization: Bearer <idToken>
// 200 OK
{
  "documents": [
    {
      "documentId": "a1b2c3d4-...",
      "filename": "technical-guide.pdf",
      "fileType": "application/pdf",
      "fileSize": 245760,
      "knowledgeDbId": "281e85f4-...",
      "chunkCount": 12,
      "createdAt": "2025-07-04T12:00:00.000000",
      "status": "processing"
    }
  ],
  "count": 1
}
GET

Get Document

/documents/{documentId}

Retrieve metadata for a specific document, including a pre-signed S3 download URL valid for 1 hour.

Path Parameters
ParameterTypeDescription
documentIdstring requiredDocument ID to retrieve.
Additional Response Fields
FieldTypeDescription
downloadUrlstringPre-signed S3 URL for downloading the file. Expires in 1 hour.
 
// 200 OK
{
  "documentId": "a1b2c3d4-...",
  "filename": "technical-guide.pdf",
  "fileType": "application/pdf",
  "fileSize": 245760,
  "knowledgeDbId": "281e85f4-...",
  "chunkCount": 12,
  "createdAt": "2025-07-04T12:00:00.000000",
  "status": "processing",
  "downloadUrl": "https://s3.amazonaws.com/..."
}
DELETE

Delete Document

/documents/{documentId}

Permanently delete a document. Removes the file from S3, its metadata sidecar, and triggers a Bedrock KB sync to remove indexed vectors.

Path Parameters
ParameterTypeDescription
documentIdstring requiredDocument ID to delete.
 
// 200 OK
{
  "message": "Document 'technical-guide.pdf' deleted successfully"
}
POST

Reindex Document

/documents/reindex

Trigger a re-processing of a document through the Bedrock Knowledge Base. Useful if indexing failed or you want to refresh the vector embeddings.

Request Body
ParameterTypeDescription
documentIdstring requiredDocument ID to reindex.
 
// 200 OK
{
  "documentId": "a1b2c3d4-...",
  "chunkCount": 0,
  "message": "Document reindex triggered. Bedrock Knowledge Base will reprocess the document."
}
