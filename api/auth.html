Authentication

KnowledgeDB uses token-based authentication. Sign up, login to receive tokens, and include the idToken as a Bearer token in all authenticated requests.

The auth flow is simple: POST /auth/signup → POST /auth/login → use the returned idToken as Authorization: Bearer <idToken>. Tokens expire in 1 hour. Use /auth/refresh to get new tokens without re-logging in.
POST

Sign Up

/auth/signup

Register a new user account. After signup, the account is immediately active — no email verification step required.

Request Body
ParameterTypeDescription
emailstring requiredUser's email address. Must be unique.
passwordstring requiredPassword. Must include uppercase, lowercase, number, and special character.
firstNamestring requiredUser's first name.
lastNamestring requiredUser's last name.
Response Fields
FieldTypeDescription
messagestringSuccess confirmation message.
userIdstringUnique user identifier (UUID).
If the email is already registered, you'll receive a 400 error with "User already exists".
// POST /auth/signup
{
  "email": "[email]",
  "password": "[password]",
  "firstName": "[name]",
  "lastName": "[name]"
}
// 200 OK
{
  "message": "User registered successfully. Please verify your email.",
  "userId": "d4f884e8-c071-7040-93cf-ce8053c7c658"
}
# cURL example
curl -X POST \
  https://fgbpqt2pq6.execute-api.us-east-1.amazonaws.com/prod/auth/signup \
  -H "Content-Type: application/json" \
  -d '{
    "email": "[email]",
    "password": "[password]",
    "firstName": "[name]",
    "lastName": "[name]"
  }'
POST

Login

/auth/login

Authenticate with email and password. Returns access, ID, and refresh tokens along with user profile information. The idToken is used for all authenticated API requests.

Request Body
ParameterTypeDescription
emailstring requiredRegistered email address.
passwordstring requiredAccount password.
Response Fields
FieldTypeDescription
accessTokenstringJWT access token.
idTokenstringJWT ID token — use this as your Bearer token.
refreshTokenstringLong-lived token for refreshing sessions.
expiresInnumberToken lifetime in seconds (3600 = 1 hour).
tokenTypestringAlways "Bearer".
userobjectUser profile: userId, email, firstName, lastName.
 
// POST /auth/login
{
  "email": "[email]",
  "password": "[password]"
}
// 200 OK
{
  "accessToken": "eyJraWQiOiJRMTNa...",
  "idToken": "eyJraWQiOiJTOXhs...",
  "refreshToken": "eyJjdHkiOiJKV1Qi...",
  "expiresIn": 3600,
  "tokenType": "Bearer",
  "user": {
    "userId": "f4789468-7021-7082-0c88-a33407efab01",
    "email": "[email]",
    "firstName": "[name]",
    "lastName": "[name]"
  }
}
# cURL example
curl -X POST \
  https://fgbpqt2pq6.execute-api.us-east-1.amazonaws.com/prod/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "[email]",
    "password": "[password]"
  }'
POST

Refresh Token

/auth/refresh

Exchange a valid refresh token for new access and ID tokens without requiring the user to re-enter credentials. Use this when your tokens expire (after 1 hour).

Request Body
ParameterTypeDescription
refreshTokenstring requiredThe refresh token received from login.
Response Fields
FieldTypeDescription
accessTokenstringNew JWT access token.
idTokenstringNew JWT ID token.
expiresInnumberToken lifetime in seconds.
tokenTypestringAlways "Bearer".
 
// POST /auth/refresh
{
  "refreshToken": "eyJjdHkiOiJKV1Qi..."
}
// 200 OK
{
  "accessToken": "eyJraWQiOiJRMTNa...",
  "idToken": "eyJraWQiOiJTOXhs...",
  "expiresIn": 3600,
  "tokenType": "Bearer"
}
POST

Logout

/auth/logout

Invalidate the current session by revoking the refresh token. Requires a valid Bearer token in the Authorization header.

Headers
HeaderValueDescription
AuthorizationBearer <idToken>Valid ID token from login.
Request Body
ParameterTypeDescription
refreshTokenstring requiredThe refresh token to revoke.
Response
FieldTypeDescription
messagestring"Successfully logged out"
 
// POST /auth/logout
// Header: Authorization: Bearer <idToken>
{
  "refreshToken": "eyJjdHkiOiJKV1Qi..."
}
// 200 OK
{
  "message": "Successfully logged out"
}
