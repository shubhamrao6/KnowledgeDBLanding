Subscription

Manage billing and subscription lifecycle. Create checkout sessions, change plans, track usage, and handle cancellations. Powered by Polar for payment processing.

All subscription endpoints (except the webhook) require authentication. Include Authorization: Bearer <idToken> in your request headers.
GET

Get Subscription

/subscription

Retrieve the current subscription status for the authenticated user, including plan details, billing period, and API key.

Response Fields (200)
FieldTypeDescription
subscriptionobjectSubscription details object.
subscription.planstringCurrent plan: free, starter, professional, or enterprise.
subscription.statusstringSubscription status: active, canceled, past_due, or trialing.
subscription.apiKeystringAPI key for x-api-key header.
subscription.currentPeriodStartstringISO 8601 start of current billing period.
subscription.currentPeriodEndstringISO 8601 end of current billing period.
subscription.createdAtstringISO 8601 subscription creation timestamp.
subscription.updatedAtstringISO 8601 last update timestamp.
 
// GET /subscription
// Header: Authorization: Bearer <idToken>

// 200 OK
{
  "subscription": {
    "plan": "starter",
    "status": "active",
    "apiKey": "IC01PUROzD70dFJtVhrVNm1AMC22kpW2...",
    "currentPeriodStart": "2026-03-09T12:05:39.906948",
    "currentPeriodEnd": "2026-04-08T12:05:39.906948",
    "createdAt": "2026-03-09T12:05:39.906948",
    "updatedAt": "2026-03-09T12:05:39.906948"
  }
}
POST

Create Checkout Session

/subscription/checkout

Create a Polar checkout session for subscribing to a plan. Returns a checkout URL that the user should be redirected to for payment.

Request Body
ParameterTypeDescription
planstring requiredPlan to subscribe to: starter, professional, or enterprise.
successUrlstring optionalURL to redirect after successful payment.
Response Fields (200)
FieldTypeDescription
checkoutUrlstringPolar checkout URL. Redirect the user here.
sessionIdstringCheckout session identifier.
 
// POST /subscription/checkout
// Header: Authorization: Bearer <idToken>
{
  "plan": "professional",
  "successUrl": "https://knowledgedb.dev/success"
}
// 200 OK
{
  "checkoutUrl": "https://polar.sh/checkout/...",
  "sessionId": "cs_a1b2c3d4e5f6"
}
POST

Change Plan

/subscription/change

Upgrade or downgrade the current subscription plan. Changes take effect immediately for upgrades, or at the end of the current billing period for downgrades.

Request Body
ParameterTypeDescription
newPlanstring requiredTarget plan: starter, professional, or enterprise.
Response Fields (200)
FieldTypeDescription
messagestringConfirmation of plan change.
previousPlanstringThe plan before the change.
newPlanstringThe new active plan.
effectiveDatestringWhen the change takes effect.
 
// POST /subscription/change
// Header: Authorization: Bearer <idToken>
{
  "newPlan": "professional"
}
// 200 OK
{
  "message": "Plan changed successfully",
  "previousPlan": "starter",
  "newPlan": "professional",
  "effectiveDate": "2026-03-09T12:00:00Z"
}
POST

Cancel Subscription

/subscription/cancel

Cancel the current subscription. By default, the subscription remains active until the end of the current billing period. Use immediate: true to cancel immediately.

Request Body
ParameterTypeDescription
immediateboolean optionalIf true, cancel immediately. Default: false (cancel at period end).
Response Fields (200)
FieldTypeDescription
messagestringCancellation confirmation.
cancelAtstringISO 8601 date when the subscription will end.
immediatebooleanWhether the cancellation was immediate.
 
// POST /subscription/cancel
// Header: Authorization: Bearer <idToken>
{
  "immediate": false
}
// 200 OK
{
  "message": "Subscription cancelled",
  "cancelAt": "2026-04-01T00:00:00Z",
  "immediate": false
}
GET

Get Usage

/subscription/usage

Retrieve usage statistics for the current billing period, including API call quotas, consumption, and rate limits.

Response Fields (200)
FieldTypeDescription
planstringCurrent subscription plan.
quotanumberTotal API call quota for the period.
usednumberNumber of API calls used.
remainingnumberRemaining API calls.
percentUsednumberPercentage of quota consumed.
periodStartstringStart date of the current billing period.
periodEndstringEnd date of the current billing period.
throttleobjectRate limiting: rateLimit (requests/sec) and burstLimit.
 
// GET /subscription/usage
// Header: Authorization: Bearer <idToken>

// 200 OK
{
  "plan": "starter",
  "quota": 10000,
  "used": 0,
  "remaining": 10000,
  "percentUsed": 0,
  "periodStart": "2026-03-01",
  "periodEnd": "2026-03-09",
  "throttle": {
    "rateLimit": 10,
    "burstLimit": 20
  }
}

POST

Webhook

/subscription/webhook

Receives subscription event callbacks from Polar. This endpoint does not require authentication — it is called directly by Polar's webhook system to notify your backend of subscription changes.

This endpoint has no authentication. It is intended to be called by Polar's servers, not by your application directly.
Request Body (from Polar)
FieldTypeDescription
typestringEvent type, e.g. subscription.created, subscription.updated, subscription.canceled.
dataobjectEvent payload containing subscription details from Polar.
Response (200)
FieldTypeDescription
receivedbooleanAlways true on success.
 
// POST /subscription/webhook
// No Authorization header required
// Called by Polar's servers
{
  "type": "subscription.updated",
  "data": {
    "id": "sub_polar_abc123",
    "status": "active",
    "product_id": "prod_xyz789",
    "customer_email": "[email]"
  }
}
// 200 OK
{
  "received": true
}
